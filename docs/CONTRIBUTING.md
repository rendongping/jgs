# 贡献指南

> 感谢你对本知识库的兴趣！本指南说明如何为“前端架构师知识图谱与学习资料库”贡献内容。

---

## 一、贡献前必读

1. 请先阅读 `docs/STYLE-GUIDE.md`，确保内容风格一致。
2. 本知识库面向**初级到高级前端工程师**，内容应兼顾易懂性和深度。
3. 所有新增内容需要经过“自检 + 审核”两步。

---

## 二、目录与命名规范

### 2.1 目录结构

```
level-0X-层级名称/
  └── 领域名称/
      ├── 01-学习文档.md
      ├── 02-练习册.md
      └── 03-面试题.md
```

### 2.2 目录命名

- 使用小写英文和连字符，如 `ai-engineering`、`node-bff`。
- 不使用拼音、中文或特殊字符。

### 2.3 文件命名

- 必须严格使用 `01-学习文档.md`、`02-练习册.md`、`03-面试题.md`。
- 保持三件套齐全。

### 2.4 领域编号

新增领域时，需要：

1. 在 README.md 的目录结构中添加对应条目。
2. 在 `前端架构师进阶规划.md` 的知识图谱中添加对应条目。
3. 在文档标题或开头标注领域编号，如 `E09 AI 工程化`。

---

## 三、内容贡献类型

### 3.1 新增知识领域

如果你认为某个领域缺失，请按以下流程：

1. 在 Issue 中提出新增建议，说明理由和拟包含内容。
2. 获得同意后，创建新目录并编写三件套文档。
3. 同步更新 README.md 和 前端架构师进阶规划.md。
4. 在 CHANGELOG.md 中记录。

### 3.2 完善现有文档

- 补充代码示例
- 修正错误或过时的内容
- 增加案例或反模式
- 优化语言表达

### 3.3 补充工具模板

在 `templates/` 或 `playbooks/` 目录下提供可直接使用的模板，如：

- ADR 模板
- 技术选型评分卡
- Code Review 清单
- 性能优化 SOP

### 3.4 更新外部资源

为每个领域的“延伸阅读”补充优质文章、书籍、视频链接。

---

## 四、内容审核流程

### 4.1 自检清单

提交前请确保：

- [ ] 新增领域三件套齐全。
- [ ] 文档符合 `STYLE-GUIDE.md` 规范。
- [ ] 代码示例可运行或经过验证。
- [ ] 没有明显的格式错误（可用 markdownlint 检查）。
- [ ] 已更新 README.md 和 前端架构师进阶规划.md。
- [ ] 已在 CHANGELOG.md 中记录变更。

### 4.2 审核维度

维护者会从以下维度审核：

| 维度 | 权重 | 说明 |
|------|------|------|
| 准确性 | 30% | 技术概念、代码示例是否正确。 |
| 完整性 | 25% | 是否覆盖核心知识点，三件套是否齐全。 |
| 易学性 | 20% | 是否通俗易懂，是否适合自学者。 |
| 一致性 | 15% | 是否符合本知识库的风格和规范。 |
| 实用性 | 10% | 是否有案例、模板或可直接落地的内容。 |

---

## 五、版本管理

### 5.1 版本号规则

采用语义化版本：

- **MAJOR**：知识体系重大重构（如新增层级、合并领域）。
- **MINOR**：新增领域、重要内容补充。
- **PATCH**：修正错误、优化表达、更新链接。

### 5.2 变更记录

所有变更必须记录在 `CHANGELOG.md` 中，格式：

```markdown
## [1.1.0] - 2026-06-18
### Added
- 新增 AI 工程化领域（E09）。
- 新增 Node.js/BFF 领域（E10）。

### Fixed
- 修正 前端架构师进阶规划.md 中的格式错误。
```

---

## 六、沟通方式

- 发现内容错误：直接提交修正，并在 CHANGELOG 中说明。
- 提出新增建议：先开 Issue 讨论。
- 重大结构调整：先在 Issue 中提案，获得共识后实施。

---

## 七、贡献者承诺

提交内容即表示你同意：

1. 内容是原创或已获得授权。
2. 内容遵循本知识库的知识共享协议。
3. 愿意接受社区审核和改进建议。

---

> 每一个贡献，都是在帮助更多前端工程师成长为架构师。感谢你的付出！
