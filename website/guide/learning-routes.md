# 前端架构师学习路线

> 本文件提供四条差异化的学习路线，帮助不同职业方向的学习者聚焦重点、少走弯路。

---

## 路线选择指南

| 路线 | 适合人群 | 核心目标 |
|------|---------|---------|
| 业务型前端架构师 | 喜欢理解业务、做中台/低代码/复杂业务系统 | 用技术赋能业务，设计可扩展的业务架构 |
| 工程型前端架构师 | 喜欢工程化、性能、质量、工具链 | 打造高效、稳定、可观测的前端工程体系 |
| 平台型前端架构师 | 喜欢跨端、可视化、基础设施、组件生态 | 建设可复用的前端平台能力 |
| 管理型前端技术负责人 | 喜欢带团队、做规划、推动组织变革 | 制定技术战略，建设团队和组织能力 |

## 路线一：业务型前端架构师

### 核心能力

- 深入理解业务领域（电商、金融、SaaS 等）。
- 设计可扩展的业务架构（DDD、BFF、中台）。
- 能主导复杂业务系统的技术方案。

### 学习重点

**基础层**：
- F01 JavaScript（L4）
- F02 TypeScript（L4）
- F03 Browser（L3）
- F04 Network（L3）
- F05 Security（L3）
- F06 HTML/CSS（L4）
- F07 a11y（L3）
- F08 Data Structures & Algorithms（L3）
- F09 Design Patterns（L3）

**工程化层**：
- E05 Design System（L4）
- E06 React 或 E07 Vue（L4）
- E10 Node.js/BFF（L4）
- E02 Monorepo（L3）
- E09 AI Engineering（L3）
- E13 Frontend SRE（L3）
- E14 Package & Supply Chain（L3）

**架构层**：
- A01 System Architecture（L5）
- A05 Data & State（L4）
- A02 Micro Frontend（L3）
- A04 Quality（L3）
- A09 Internationalization（L3）
- A10 Visualization & Graphics（L3）
- A12 Data Engineering（L4）

**领导力层**：
- L01 Business（L5）
- L03 Strategy（L4）
- L02 Team（L3）
- L04 Communication（L4）
- L05 Project Management（L3）
- L06 Hiring（L3）
- L07 Tech Brand & Community（L3）
- L08 Tech Governance（L3）

### 通关项目

1. 搭建一个可复用的业务组件库。
2. 设计并实现一个复杂表单的低代码配置方案。
3. 主导一个 BFF 架构升级项目。
4. 输出一份业务领域模型设计文档。

## 通用学习节奏建议

| 时间 | 行动 |
|------|------|
| 每周 | 精读一个领域的学习文档 |
| 每月 | 完成对应练习册 + 面试题 |
| 每季度 | 做一次自评，调整学习重点 |
| 每半年 | 完成一个通关项目 |
| 每年 | 参与或主导一个中大型项目 |

---

## 学习路线与能力金字塔的关系

无论选择哪条路线，都必须先夯实基础层，再逐步向上攀登。不同路线在工程化层、架构层、领导力层的侧重点不同，但都需要有一定广度。

```
        ┌─────────────────────────────┐
        │     领导力层（路线差异化）      │
        ├─────────────────────────────┤
        │     架构层（路线差异化）        │
        ├─────────────────────────────┤
        │   工程化层（路线差异化）        │
        ├─────────────────────────────┤
        │   基础层（所有路线共用）        │
        └─────────────────────────────┘
```
