# 架构师工作流

前端架构师的工作不只是写代码，更包括技术决策、团队建设、战略规划等。

本页面简要介绍架构师的典型工作流，详细内容请查看 [架构师工作流与面试模拟](/resources/architect-workflow)。

## 典型一周

| 时间 | 重点工作 |
|------|---------|
| 周一 | 业务对齐、团队同步、技术雷达更新 |
| 周二 | 需求评审、架构设计、Code Review |
| 周三 | 性能优化、可观测性、技术调研 |
| 周四 | 工程化落地、组件库/工具链升级、技术文档 |
| 周五 | 事故复盘、技术债清理、1on1 沟通 |

## 核心工作模式

### 需求到来时

```
业务需求 → 理解目标 → 评估可行性 → 技术方案 → 评审 → 落地
```

### 技术选型时

```
明确问题 → 候选方案 → 评估标准 → POC → 决策 → ADR
```

### 线上故障时

```
发现 → 确认影响 → 止损 → 定位 → 修复 → 验证 → 复盘
```

## 关键能力

- 技术深度与广度
- 架构设计与决策
- 业务理解与技术赋能
- 团队管理与沟通影响
- 技术战略与长期规划
