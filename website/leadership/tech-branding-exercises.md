# 技术品牌与社区影响力练习册

D. 降低工资

开源项目成功的关键不包括？

A. 解决真实问题  
B. 完善的文档

D. 活跃的社区

### 第 3 题（🟡）

雇主品牌建设最重要的是？

A. 高薪  
B. 真实的技术文化和工作体验

### 第 4 题（🟡）

技术演讲准备的第一步是？

A. 写 PPT  
B. 明确听众和核心观点  
C. 找模板  
D. 报名大会

### 第 5 题（🔴）

衡量开源项目健康度最常用的指标是？

A. 代码行数  
B. contributors、issue 响应、release 频率  
C. README 字数  
D. 项目名长度

### 第 6 题

**参考计划**：
- 第 1 月：确定定位、搭建博客、产出 4 篇深度文章。
- 第 2 月：建立内容日历、鼓励团队成员投稿、多平台分发。
- 第 3 月：投稿外部技术媒体、策划一次线上分享。

### 第 7 题

::: details 查看答案与解析
**参考思路**：
- 原因：贡献门槛高、文档不足、Issue 响应慢、缺乏引导。
- 改进：完善贡献指南、标注 good-first-issue、及时响应、举办贡献者活动。
:::

### 第 8 题

**参考设计**：
- 议程：主题演讲 + 业务案例 + 开源实践 + 圆桌讨论。
- 嘉宾：内部专家 + 行业 KOL + 社区贡献者。
- 宣传：多渠道预热、会后内容沉淀。

### 第 9 题

**参考计划**：
- 博客：每月 2 篇，覆盖技术深度文章和踩坑记录。
- 演讲：每季度 1-2 次内外部分享。
- 开源：每年开源 1-2 个工具，维护核心项目。

### 第 10 题

**参考模型**：
- 角色：Maintainer、Committer、Contributor、User。
- 决策：重大变更通过 RFC 讨论，Maintainer 投票。
- 发布：语义化版本 + changelog + 发布说明。

### 第 11 题

**参考路线图**：
- 第 1-2 月：梳理技术亮点，建立博客和公众号。
- 第 3-4 月：持续输出内容，参与外部社区。
- 第 5-6 月：举办技术活动，开源内部工具，申请大会演讲。
