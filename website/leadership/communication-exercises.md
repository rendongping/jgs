## 参考答案

### 第 1 题

**答案：B**

向上汇报应结论先行，高管时间宝贵，先讲核心结论再补充细节。

### 第 2 题

**答案：A**

RACI 中 A 是 Accountable（最终负责人）。

### 第 3 题

**答案：B**

找到共同目标是跨部门沟通的基础，能化解对立，促进协作。

### 第 4 题

**答案：B**

技术写作应采用金字塔原理，结论先行，再用论据支撑。

### 第 5 题

**答案：B**

合作（Collaborating）是高关注自己、高关注他人的双赢策略。

### 第 6 题

**参考思路**：
1. 用业务语言重述方案价值："首屏从 3 秒降到 1.5 秒，预计跳出率下降 10%。"
2. 提供数据支撑：竞品对比、用户流失数据。
3. 说明不做的成本：技术债累积、未来重构成本。
4. 提出最小可行方案：先做一个 MVP 验证效果。
5. 邀请产品经理一起设定成功指标。

### 第 7 题

**参考思路**：
1. 分别与两人单独沟通，了解各自的真实顾虑。
2. 组织正式的技术评审会，聚焦问题而非人。
3. 列出选型标准（性能、成本、团队能力、维护性）。
4. 要求双方用数据和 POC 支撑观点。
5. 如果无法一致，作为负责人做出决策并承担后果。
6. 会后强调"可以反对决策，但必须执行"。

### 第 8 题

**参考汇报结构**：

```
结论：项目可能延期两周上线。

关键信息：
1. 风险原因：第三方 SDK 接口变更，需要重新集成。
2. 影响范围：影响用户登录模块，预计影响 30% 用户。
3. 当前进展：已联系第三方确认变更点，正在评估工作量。

应对方案：
A. 等待第三方修复（风险高，时间不可控）
B. 自行适配新接口（可控，但增加两周）
C. 暂时回退到旧版本（风险低，但缺失新功能）
推荐方案：B

需要的支持：申请增加 1 名后端同学支援，预算审批。

下一步：今日内确定方案，本周五更新进展。
```

### 第 9 题

**演讲大纲**：

```
开场（2min）：一个真实案例，慢页面导致的用户流失
问题（5min）：性能指标 LCP/INP/CLS 是什么？当前项目问题
方案（15min）：
  - 资源优化：图片、代码分割
  - 渲染优化：SSR、关键 CSS
  - 运行时优化：减少重排、缓存策略
行动（5min）：团队可以立即做的 3 件事
结尾（3min）：总结 + Q&A
```

### 第 10 题

**RFC 评审会流程**：

1. 会前 2 天分发 RFC，要求参会者提前阅读并评论。
2. 会议开始：作者 10 分钟讲解核心内容。
3. 逐条讨论评论区问题，每个问题限时 5 分钟。
4. 对方案、风险、实施计划进行投票或决策。
5. 会议结束前明确：通过/修改后通过/驳回，以及下一步 owner。
6. 会后 24 小时内发送会议纪要和更新后的 RFC。

### 第 11 题

**90 天沟通计划**：

**第 1-30 天：倾听与建立信任**
- 与每个成员 1:1，了解期望和痛点。
- 不急于推翻原有做法。
- 公开承认团队的贡献。

**第 31-60 天：小胜与共识**
- 推动 1-2 个能快速见效的改进。
- 与团队共同制定新的工作规范。
- 让成员参与决策，增强归属感。

**第 61-90 天：文化与长期规划**
- 建立定期的团队会议和 1:1 节奏。
- 制定团队技术路线图。
- 表彰优秀表现，建立正向反馈循环。
