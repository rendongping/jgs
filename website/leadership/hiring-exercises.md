## 参考答案

### 第 1 题

**答案：B**

结构化面试通过统一问题和评分标准，减少主观偏见。

### 第 2 题

**答案：B**

SBI 是 Situation-Behavior-Impact，B 是 Behavior（行为）。

### 第 3 题

**答案：B**

代码作业通常控制在 2-4 小时，既能考察能力，又不过度占用候选人时间。

### 第 4 题

**答案：B**

接班人计划能降低关键岗位人员变动带来的风险。

### 第 5 题

**答案：B**

心理安全是高绩效团队的核心特征，成员敢于表达和试错。

### 第 6 题

**参考面试问题清单**：

**技术基础（20min）**
- 解释事件循环和微任务。
- 如何实现一个深拷贝？
- React hooks 的使用规则和常见坑。

**项目经验（20min）**
- 讲一个你最有挑战的项目，你的角色和贡献是什么？
- 项目中遇到的最大技术冲突是什么？如何解决的？
- 如果重来一次，你会做哪些不同决策？

**软技能（15min）**
- 你如何与产品经理沟通不合理的需求？
- 你如何指导初级工程师成长？
- 描述一次你推动团队改进流程的经历。

### 第 7 题

**参考思路**：
1. 安排非正式的 1:1，表达重视和感谢。
2. 了解离职的真实原因（薪酬、成长、氛围、外部机会）。
3. 如果是内部可解决的问题，提出具体方案：
   - 薪酬调整或股权激励
   - 新的项目挑战或晋升机会
   - 工作方式调整
4. 如果是不可逆原因，祝福并建立 alumni 关系。
5. 做好交接计划，确保知识转移。
6. 分析离职根因，优化团队管理。

### 第 8 题

**职业发展通道示例**：

```
专业通道：
初级工程师 → 中级工程师 → 高级工程师 → 技术专家 → 首席工程师

管理通道：
高级工程师 → 技术组长 → 前端负责人 → 技术总监 → VP of Engineering

每条通道明确：
- 能力要求
- 代表作/成果要求
- 评审标准
- 晋升周期
```

### 第 9 题

**代码作业示例**：

**题目**：实现一个带搜索、排序、分页的联系人列表页面。

**要求**：
- 使用 React 或 Vue。
- 数据用 mock API。
- 实现单元测试。
- 编写 README 说明运行方式。

**评分标准**：
- 功能完整（30%）
- 代码质量与规范（25%）
- 组件设计合理性（20%）
- 测试覆盖（15%）
- 文档与可运行性（10%）

### 第 10 题

**绩效管理流程**：

**目标设定（年初）**：
- 与员工共同制定 OKR 和成长目标。
- 明确衡量标准和权重。

**持续反馈（日常）**：
- 每周 1:1 沟通进展和困难。
- 及时表扬和纠正。

**中期 review（年中）**：
- 评估目标完成进度。
- 调整不合理目标。
- 制定下半年重点。

**年终评估（年末）**：
- 自评 + 主管评价 + 360 度反馈。
- 综合结果、过程、成长。
- 确定绩效等级和激励。

**结果应用**：
- 薪酬调整
- 晋升/轮岗
- 培训计划

### 第 11 题

**培训体系框架**：

**新人培训（入职 1-3 个月）**
- 公司文化与价值观
- 前端技术栈与开发规范
- 代码仓库与 CI/CD 流程
- 导师制辅导

**持续学习**
- 每周技术分享（轮值主讲）
- 每月读书会
- 每季度技术大会/外部培训

**专项提升**
- 架构师训练营
- 管理技能培训
- 软技能培训（沟通、演讲）

**知识沉淀**
- 内部 Wiki
- 技术博客
- 复盘案例库
