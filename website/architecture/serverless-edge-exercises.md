## 参考答案

### 第 1 题

**答案：B**

Serverless 无需管理服务器，按调用次数和资源使用付费。

### 第 2 题

**答案：B**

Edge 计算在靠近用户的节点运行，延迟更低。

### 第 3 题

**答案：B**

边缘 SSR 和 A/B 测试是 Edge 函数的典型场景。

### 第 4 题

**答案：B**

Edge 函数执行时间和 CPU/内存资源受限。

### 第 5 题

**答案：D**

减少依赖可降低启动时间，预留实例/保持活跃可避免冷启动。

### 第 6 题

**参考思路**：
- 使用 Edge 函数（如 Cloudflare Workers 或 Vercel Edge）。
- 根据请求头中的地理位置或 IP 返回不同内容。
- 结合 CDN 缓存策略。

### 第 7 题

**参考思路**：
- 使用 Edge SSR，在边缘节点渲染首屏。
- 将静态资源缓存到 CDN。
- 个性化内容通过 Edge 函数动态注入。

### 第 8 题

**参考思路**：
- Edge 函数拦截所有请求。
- 鉴权：校验 Token 或 Cookie。
- 限流：基于 IP 或用户 ID 计数。
- 灰度：按规则分流到不同版本。

### 第 9 题

**参考对比**：
- Vercel Edge：Next.js 生态集成好，适合 SSR/ISR。
- Cloudflare Workers：全球节点多、性能强，适合网关和 Workers。
- AWS Lambda@Edge：与 AWS 生态集成，适合已有 CloudFront 架构。

### 第 10 题

**参考思路**：
- 静态资源通过多区域 CDN 分发。
- Edge 函数根据用户区域返回本地化内容。
- 数据层使用多区域数据库或缓存。
- 监控各区域延迟和可用性。

### 第 11 题

**参考思路**：
- 使用 JWT 等无状态 Token。
- 将会话数据存储在分布式 KV（如 Cloudflare KV、DynamoDB）。
- 避免在 Edge 函数中保存本地状态。
