## 一、基础题

### 1. 什么是语义化 HTML？为什么要语义化？

**参考答案**：

语义化 HTML 是指使用恰当的 HTML 标签表达内容的含义，而不仅仅是呈现效果。

主要收益：
- **可访问性**：屏幕阅读器依赖语义标签进行导航。
- **SEO**：搜索引擎更容易理解页面结构。
- **可维护性**：代码更易读，结构更清晰。
- **默认行为**：`<button>`、`<a>` 等标签自带键盘交互和焦点管理。

**评分维度**：
- 说出语义化的定义（40%）
- 列举至少 3 个收益（40%）
- 能举例说明（20%）

### 2. 解释 BEM 命名规范，并说明其优缺点。

**参考答案**：

BEM 是 Block-Element-Modifier 的缩写，通过命名约定组织 CSS 类名：
- Block：独立模块，如 `.card`
- Element：块的组成部分，如 `.card__title`
- Modifier：状态或变体，如 `.card--featured`

优点：
- 命名即作用域，降低冲突。
- 无需工具即可实现模块化。

缺点：
- 类名较长。
- 深层嵌套组件表达力有限。

**评分维度**：
- 解释 BEM 三部分（40%）
- 说出优点（30%）
- 说出缺点并给出替代方案（30%）

### 3. CSS 中 `em`、`rem`、`px`、`vw` 有什么区别？如何选择？

**参考答案**：

| 单位 | 参考基准 | 适用场景 |
|------|---------|---------|
| px | 物理像素 | 边框、阴影等不需要缩放的场景 |
| em | 父元素字体大小 | 需要相对父级缩放的场景 |
| rem | 根元素字体大小 | 字体大小、间距，便于整体缩放 |
| vw/vh | 视口宽高 | 全屏布局、流体排版 |

选择建议：
- 字体和间距优先使用 `rem`。
- 边框使用 `px`。
- 响应式布局可结合 `clamp()` 使用 `vw`。

**评分维度**：
- 区分四个单位（50%）
- 给出选择建议（30%）
- 提到 `clamp()` 等现代方案（20%）

### 4. 什么是 CSS 优先级？如何计算？

**参考答案**：

CSS 优先级由选择器的特殊性决定，从高到低：
1. 内联样式（1000）
2. ID 选择器（100）
3. 类选择器、属性选择器、伪类（10）
4. 元素选择器、伪元素（1）

`!important` 可以覆盖优先级，但应尽量避免。

当优先级相同时，后声明的规则生效。

**评分维度**：
- 说出优先级计算规则（50%）
- 解释 `!important` 的影响（25%）
- 提到层叠顺序（25%）

### 5. 如何实现一个三栏布局，中间自适应？

**参考答案**：

**方案一：Flexbox**

```css
.layout {
  display: flex;
}
.layout__side {
  width: 200px;
}
.layout__main {
  flex: 1;
}
```

**方案二：Grid**

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
}
```

Grid 更适合二维布局，Flexbox 更适合一维布局。

**评分维度**：
- 写出 Flexbox 实现（40%）
- 写出 Grid 实现（30%）
- 说明两种方案适用场景（30%）

## 二、进阶题

### 6. 什么是 CSS 工程化？前端项目中为什么需要 CSS 工程化？

**参考答案**：

CSS 工程化是指建立一套可预测、可复用、可度量的样式生产体系，解决大型项目中的样式管理问题。

需要工程化的原因：
- 全局命名冲突
- 选择器优先级战争
- 样式冗余与包体积膨胀
- 主题切换困难
- 响应式逻辑分散
- 可访问性合规要求

工程化手段包括：BEM、ITCSS、CSS Modules、CSS-in-JS、Atomic CSS、Design Token、Stylelint 等。

**评分维度**：
- 定义 CSS 工程化（30%）
- 列举痛点（40%）
- 列举工程化手段（30%）

### 7. 解释一下容器查询（Container Query）及其使用场景。

**参考答案**：

容器查询允许组件根据其所在容器的大小调整样式，而不是基于整个视口。

```css
.card-wrapper {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}
```

使用场景：
- 同一个组件在不同布局中表现不同。
- 侧边栏、网格项等局部容器需要响应式。
- 组件库中需要组件级响应式能力。

**评分维度**：
- 解释容器查询原理（40%）
- 写出基本语法（30%）
- 给出使用场景（30%）

### 8. 如何减少 CSS 引起的重排和重绘？

**参考答案**：

减少重排：
- 避免频繁读写会触发布局的属性（width、height、top、margin）。
- 批量修改样式时，先读取再写入，或使用 `requestAnimationFrame`。
- 使用 `transform` 和 `opacity` 做动画。

减少重绘：
- 避免频繁修改颜色、背景等外观属性。
- 使用 `will-change` 提示浏览器优化（慎用）。
- 提升独立渲染层，减少整体重绘范围。

**评分维度**：
- 区分重排和重绘（30%）
- 说出减少重排的方法（40%）
- 说出减少重绘的方法（30%）

### 9. Design Token 是什么？为什么现代设计系统需要它？

**参考答案**：

Design Token 是设计系统中的最小原子，如颜色、字体、间距、圆角等，以代码可消费的形式存在。

为什么需要：
- 统一设计与开发之间的语言。
- 支持主题切换（浅色/深色/多品牌）。
- 避免硬编码，减少样式冗余。
- 便于工具链集成（Figma、Storybook、代码）。

Token 通常分层：
- 原始 Token：`--color-blue-500`
- 语义 Token：`--color-brand-primary`
- 组件 Token：`--button-bg-primary`

**评分维度**：
- 解释 Design Token 概念（30%）
- 说明其价值（40%）
- 说明 Token 分层（30%）

## 三、高级题

### 10. 设计一个支持多主题的组件库样式体系。

**参考答案要点**：

1. **Token 层**：定义原始 Token 和语义 Token，使用 CSS 变量。
2. **主题切换**：通过切换根元素的 `data-theme` 属性修改变量值。
3. **组件层**：组件只引用语义 Token，不直接引用原始颜色。
4. **构建层**：使用 PostCSS 或构建工具处理变量回退。
5. **文档与工具**：在 Storybook 中提供主题切换器。

```css
:root {
  --color-primary: #3b82f6;
}

[data-theme="dark"] {
  --color-primary: #60a5fa;
}

.btn-primary {
  background: var(--color-primary);
}
```

**评分维度**：
- 设计 Token 分层（30%）
- 说明主题切换机制（30%）
- 考虑组件与工具链集成（25%）
- 表达清晰、有扩展性（15%）

### 11. 你如何评估和重构一个存在大量技术债的样式代码库？

**参考答案要点**：

1. **现状评估**：统计样式文件大小、重复规则、`!important` 数量、选择器平均深度。
2. **建立基线**：用 Lighthouse、Stylelint、PurgeCSS 等工具量化问题。
3. **制定规范**：命名约定、分层架构、Token 体系。
4. **分阶段重构**：
   - 第一阶段：禁止新增 `!important`，引入变量。
   - 第二阶段：拆分文件，建立组件样式。
   - 第三阶段：响应式统一，主题化。
5. **验证与监控**：自动化检查、视觉回归测试、性能指标跟踪。

**评分维度**：
- 评估方法（25%）
- 规范制定（25%）
- 分阶段方案（35%）
- 验证机制（15%）

### 12. 在前端性能优化中，CSS 加载策略有哪些最佳实践？

**参考答案要点**：

1. **关键 CSS 内联**：将首屏所需 CSS 直接写入 `<head>`，减少渲染阻塞。
2. **异步加载非关键 CSS**：
   - `<link rel="preload" as="style" href="non-critical.css">`
   - `<link rel="stylesheet" href="non-critical.css" media="print" onload="this.media='all'">`
3. **避免 @import**：串行加载会阻塞解析。
4. **压缩与去重**：Minify、PurgeCSS、CSS 分割。
5. **利用缓存**：为 CSS 文件设置长期缓存，配合文件名 hash。
6. **减少重排重绘**：动画使用 `transform`/`opacity`。

**评分维度**：
- 关键 CSS 策略（25%）
- 异步加载方案（25%）
- 构建优化（25%）
- 运行时性能（25%）
