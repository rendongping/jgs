## 参考答案

### 第 1 题

::: details 查看答案与解析
**答案：B**

S 是 Single Responsibility Principle，单一职责原则。
:::

### 第 2 题

::: details 查看答案与解析
**答案：B**

Vue 3 使用 Proxy 实现响应式，属于代理模式。
:::

### 第 3 题

::: details 查看答案与解析
**答案：A**

Redux 的订阅-发布机制属于观察者模式。
:::

### 第 4 题

::: details 查看答案与解析
**答案：B**

策略模式用于封装可互相替换的算法或行为。
:::

### 第 5 题

::: details 查看答案与解析
**答案：B**

HOC 在不修改原组件的情况下增强功能，本质是装饰器模式。
:::

### 第 6 题

::: details 查看答案与解析
**参考思路**：
- 违反单一职责原则（SRP）。
- 重构：将 UI 展示、数据请求、状态管理、业务逻辑拆分到不同模块或 Hook。
- 使用容器/展示组件模式。
:::

### 第 7 题

::: details 查看答案与解析
**参考思路**：
- 使用策略模式。
- 定义校验器接口，每种规则实现一个校验器。
- 运行时组合校验规则。
:::

### 第 8 题

::: details 查看答案与解析
**参考思路**：
- 定义组件注册接口。
- 通过配置或插件机制注册新组件。
- 渲染流程依赖抽象接口，不依赖具体组件。
:::

### 第 9 题

::: details 查看答案与解析
**参考实现**：

```js
class EventBus {
  constructor() { this.events = {}; }
  on(event, fn) {
    (this.events[event] ||= []).push(fn);
  }
  emit(event, ...args) {
    (this.events[event] || []).forEach(fn => fn(...args));
  }
  off(event, fn) {
    if (!this.events[event]) return;
    this.events[event] = this.events[event].filter(f => f !== fn);
  }
}
```
:::

### 第 10 题

::: details 查看答案与解析
**参考思路**：
- 定义 Theme 抽象接口。
- 每个主题实现该接口。
- 通过 ThemeProvider 注入当前主题。
- 组件通过上下文消费主题，新增主题只需添加实现类。
:::

### 第 11 题

::: details 查看答案与解析
**参考思路**：
- 使用策略模式定义不同埋点策略。
- 使用观察者模式监听 DOM 事件。
- 使用命令模式封装埋点命令，支持队列和重试。
- 通过配置中心动态启用/禁用策略。
:::
