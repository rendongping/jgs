# F08 数据结构与算法

> 目标：掌握前端开发中常用的数据结构与算法，提升代码效率、面试竞争力与复杂问题解决能力。

---

## 核心要点（TL;DR）

- 数据结构是算法的基础，选择合适的数据结构能把时间复杂度从 O(n) 降到 O(1) 或 O(log n)。
- 前端高频数据结构：数组、栈、队列、链表、哈希表、树、图、堆。
- 前端高频算法：排序、双指针、滑动窗口、递归/回溯、BFS/DFS、动态规划、贪心。
- 复杂度分析是评估方案的核心工具，必须熟练掌握时间/空间复杂度。
- 算法能力决定了你能否写出高性能、可扩展的前端代码。

---

## 1. 复杂度分析

### 1.1 时间复杂度

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 哈希表查询 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 快速排序、归并排序 |
| O(n²) | 平方 | 冒泡排序、双重循环 |
| O(2ⁿ) | 指数 | 递归回溯 |

### 1.2 空间复杂度

- 额外空间：算法运行过程中申请的内存。
- 原地算法：空间复杂度为 O(1)。

## 2. 常用数据结构

### 2.1 数组与字符串

- 连续内存，支持随机访问 O(1)。
- 前端应用：虚拟列表、滑动窗口、去重、扁平化。

### 2.2 栈与队列

- **栈**：后进先出，用于函数调用、撤销操作、括号匹配。
- **队列**：先进先出，用于任务调度、BFS、消息队列。

### 2.3 链表

- 非连续内存，插入删除 O(1)，查找 O(n)。
- 前端应用：React Fiber 链表结构、LRU 缓存。

### 2.4 哈希表

- 键值对存储，查询 O(1)。
- 前端应用：Map/Set、缓存、索引、去重。

### 2.5 树

- **二叉树**：遍历（前序、中序、后序、层序）。
- **二叉搜索树**：查找 O(log n)。
- **堆**：优先队列、Top K。
- **Trie 树**：前缀匹配、自动补全。
- 前端应用：DOM 树、虚拟 DOM Diff、组件树。

### 2.6 图

- 节点与边的集合。
- 遍历：BFS、DFS。
- 前端应用：依赖图、知识图谱、流程图、布局算法。

## 3. 常用算法

### 3.1 排序算法

| 算法 | 平均复杂度 | 稳定性 | 适用场景 |
|------|-----------|--------|---------|
| 快速排序 | O(n log n) | 不稳定 | 通用 |
| 归并排序 | O(n log n) | 稳定 | 链表、外部排序 |
| 堆排序 | O(n log n) | 不稳定 | Top K |
| 计数排序 | O(n + k) | 稳定 | 数据范围小 |

### 3.2 双指针与滑动窗口

- 双指针：快慢指针、左右指针。
- 滑动窗口：子数组、子串问题。

### 3.3 递归与回溯

- 递归：分解问题为子问题。
- 回溯：DFS + 剪枝，用于组合、排列、N 皇后。

### 3.4 动态规划

- 将复杂问题拆分为子问题，保存中间结果。
- 经典问题：斐波那契、背包、最长公共子序列、股票买卖。

### 3.5 贪心算法

- 每一步选择当前最优解。
- 适用场景：区间调度、 Huffman 编码、最小生成树。

## 4. 前端场景题

### 4.1 虚拟列表

- 问题：渲染 10 万条数据卡顿。
- 方案：只渲染可视区域数据，使用滑动窗口思想。

### 4.2 LRU 缓存

- 问题：实现最近最少使用缓存。
- 方案：Map + 双向链表。

### 4.3 防抖与节流

- 本质：时间维度上的队列/窗口控制。

### 4.4 路由匹配

- 问题：实现前端路由的通配符匹配。
- 方案：Trie 树或正则。

## 5. 学习建议

- 先掌握复杂度和基础数据结构。
- 按题型刷题：数组→链表→树→递归→动态规划→图。
- 每道题都要分析时间/空间复杂度。
- 结合前端源码学习：React Fiber、Vue 响应式、Webpack 依赖图。

## 常见误区

| 误区 | 正确理解 |
|------|---------|
| 只背算法，不分析复杂度 | 复杂度是评估方案的核心 |
| 追求最优解，忽略可读性 | 先正确，再优化 |
| 忽视数据结构选型 | 合适的数据结构比算法更重要 |
| 只做 LeetCode，不结合前端 | 要从业务场景出发 |
