### 1. 什么是时间复杂度和空间复杂度？

**参考答案**：

时间复杂度描述算法执行时间随输入规模增长的变化趋势，空间复杂度描述算法额外占用内存随输入规模增长的变化趋势。通常使用大 O 表示法。

常见复杂度量级：O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n) < O(n!)。

分析时需区分最好情况、最坏情况和平均情况。例如快速排序最好 O(n log n)、最坏 O(n^2)。

**评分维度**：

**常见错误**：
- 混淆时间复杂度和实际运行时间（前者是增长趋势，后者受硬件和输入影响）
- 忽略常数和低阶项后直接比较复杂度（如 100n 与 0.1n^2 在 n 较小时需具体分析）
- 递归函数复杂度分析错误（忘记考虑递归调用次数）

### 2. 数组和链表的区别是什么？

**参考答案**：

- **数组**：连续内存空间，随机访问 O(1)，插入删除 O(n)（需移动元素）。JS 中数组为动态数组，扩容时触发 O(n) 拷贝。
- **链表**：非连续内存，通过指针连接。随机访问 O(n)（需遍历），插入删除 O(1)（已知节点位置时）。

链表分类：单向链表、双向链表、循环链表。JS 中无原生链表实现，需手动构建（{val, next} 结构）。

前端场景：React 的 Fiber 架构使用链表实现可中断的协调；浏览器历史记录可用双向链表实现。

**常见错误**：

- 混淆单向链表和双向链表的区别
- 认为 JS 数组底层总是真正的连续内存数组（V8 中有快数组和慢数组之分）

### 4. 栈（Stack）与队列（Queue）的原理及区别

**问题描述**：请阐述栈和队列的核心特性、各自的实现方式，并说明它们在前端开发中的典型应用场景。

**参考答案**：

**栈（Stack）**：后进先出（LIFO）。操作包括 push（入栈）、pop（出栈）、peek（查看栈顶）。可用数组或链表实现。

**队列（Queue）**：先进先出（FIFO）。操作包括 enqueue（入队）、dequeue（出队）、front（查看队首）。

JavaScript 实现示例：

class Stack {
  constructor() { this.items = []; }
  push(val) { this.items.push(val); }
  pop() { return this.items.pop(); }
  peek() { return this.items[this.items.length - 1]; }
  isEmpty() { return this.items.length === 0; }
}

class Queue {
  constructor() { this.items = []; }
  enqueue(val) { this.items.push(val); }
  dequeue() { return this.items.shift(); }
  front() { return this.items[0]; }
  isEmpty() { return this.items.length === 0; }
}

前端应用场景：
- 栈：函数调用堆栈、Undo/Redo 操作、浏览器前进后退
- 队列：事件循环（Event Loop）中的任务队列、BFS 遍历、消息队列
- 双端队列（Deque）：滑动窗口最大值问题

注意：直接用数组实现队列时 shift() 为 O(n)，生产环境建议用链表或两个栈优化。

- 原理阐述（30%）
- 代码实现（30%）

- 应用场景（20%）

**常见错误**：
- 混淆 LIFO 和 FIFO
- 认为 JS 数组实现队列的 shift() 是 O(1)（实际为 O(n) 因为需要移动元素）
- 忽略队列在事件循环中的应用

### 5. 反转链表

**问题描述**：给定一个单向链表的头节点，将其反转，返回反转后的新头节点。

**参考答案**：

反转链表是链表操作中最基础也最常考的问题，有迭代和递归两种解法。

**解法一：迭代法**

function reverseList(head) {
  let prev = null;
  let curr = head;
  while (curr !== null) {
    const next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
  }
  return prev;
}

时间复杂度 O(n)，空间复杂度 O(1)。

**解法二：递归法**

function reverseList(head) {
  if (head === null || head.next === null) return head;
  const newHead = reverseList(head.next);
  head.next.next = head;
  head.next = null;
  return newHead;
}

时间复杂度 O(n)，空间复杂度 O(n)（递归调用栈）。

关键思路：迭代法用三个指针（prev, curr, next）维护反转过程；递归法先反转后续链表，再将当前节点的 next 指向自己。

**评分维度**：

**常见错误**：
- 迭代法中忘记保存 next 指针导致链表断裂
- 递归法终止条件错误或忘记处理空链表
- 返回值不是新的头节点（迭代法的 prev 或递归法的 newHead）

### 6. 如何用栈实现队列？

**问题描述**：使用两个栈实现一个队列，支持 push（入队）、pop（出队）、peek（查看队首）和 empty（判空）操作。

**参考答案**：

使用两个栈 stackIn 和 stackOut。push 时直接压入 stackIn；pop/peek 时，若 stackOut 为空则将 stackIn 全部弹出并压入 stackOut，再从 stackOut 弹出/查看。

class MyQueue {
  constructor() {
    this.stackIn = [];
    this.stackOut = [];
  }
  push(x) {
    this.stackIn.push(x);
  }
  pop() {
    if (this.stackOut.length === 0) {
      while (this.stackIn.length) {
        this.stackOut.push(this.stackIn.pop());
      }
    }
    return this.stackOut.pop();
  }
  peek() {
    const val = this.pop();
    this.stackOut.push(val);
    return val;
  }
  empty() {
    return this.stackIn.length === 0 && this.stackOut.length === 0;
  }
}

均摊时间复杂度：每个元素入队和出队各操作常数次，均摊 O(1)。空间复杂度 O(n)。

**评分维度**：
- 思路清晰（40%）

**常见错误**：
- 未在 pop 之前检查 stackOut 是否为空，导致顺序错乱
- 忽略 peek 复用 pop 逻辑后需将元素放回 stackOut
- 空间复杂度分析错误（误认为是 O(1)）

### 7. 解释二叉树的前序、中序、后序遍历，并说明应用场景。

**问题描述**：请说明二叉树三种深度优先遍历的规则、代码实现和实际应用场景。

**参考答案**：

- **前序（Preorder）**：根 -> 左 -> 右，用于复制树、序列化树结构。
- **中序（Inorder）**：左 -> 根 -> 右，BST 中序遍历得到升序序列。
- **后序（Postorder）**：左 -> 右 -> 根，用于释放节点、计算树的高度。

递归实现：

function preorder(root) {
  if (!root) return;
  console.log(root.val);
  preorder(root.left);
  preorder(root.right);
}

function inorder(root) {
  if (!root) return;
  inorder(root.left);
  console.log(root.val);
  inorder(root.right);
}

function postorder(root) {
  if (!root) return;
  postorder(root.left);
  postorder(root.right);
  console.log(root.val);
}

迭代实现（以中序为例）：

function inorderTraversal(root) {
  const stack = [];
  const res = [];
  let curr = root;
  while (curr || stack.length) {
    while (curr) {
      stack.push(curr);
      curr = curr.left;
    }
    curr = stack.pop();
    res.push(curr.val);
    curr = curr.right;
  }
  return res;
}

时间复杂度 O(n)，空间复杂度 O(n)。

**评分维度**：
- 三种遍历规则（50%）
- 代码或递归过程（30%）
- 应用场景（20%）

**常见错误**：
- 混淆前序和后序的处理顺序
- 迭代实现中忘记维护遍历指针
- 未理解三种遍历在实际场景中的用途差异

### 8. LRU 缓存机制

**问题描述**：设计一个 LRU（最近最少使用）缓存，支持 get(key) 和 put(key, value) 操作，get 和 put 的平均时间复杂度均为 O(1)。缓存容量有限，超出容量时淘汰最久未使用的数据。

**参考答案**：

LRU 缓存的核心数据结构是哈希表 + 双向链表。哈希表提供 O(1) 的查找，双向链表维护数据的使用顺序。

class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }
  get(key) {
    if (!this.cache.has(key)) return -1;
    const value = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, value);
    return value;
  }
  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key);
    } else if (this.cache.size >= this.capacity) {
      const oldestKey = this.cache.keys().next().value;
      this.cache.delete(oldestKey);
    }
    this.cache.set(key, value);
  }
}

说明：利用 Map 的插入顺序特性，最近使用的 key 重新插入后会在最末尾，Map.keys().next().value 取到最久未使用的 key。

（面试中也可手动实现双向链表 + 哈希表，以展示对数据结构的理解）

**常见错误**：

### 9. 二叉树的最近公共祖先（LCA）

**问题描述**：给定一棵二叉树和两个节点 p、q，找到它们最近的公共祖先节点。最近公共祖先指两个节点在树中最低的（即最深的）共同祖先节点。

**参考答案**：

递归解法：从根节点向下搜索，若当前节点为 null 或等于 p 或 q 则直接返回。递归搜索左右子树：

function lowestCommonAncestor(root, p, q) {
  if (root === null || root === p || root === q) return root;
  const left = lowestCommonAncestor(root.left, p, q);
  const right = lowestCommonAncestor(root.right, p, q);
  if (left && right) return root;
  return left || right;
}

核心思路：
1. 如果 p 和 q 分别在当前节点的左右子树中，则当前节点即为 LCA。
2. 如果都在左子树，则 LCA 在左子树中。
3. 如果都在右子树，则 LCA 在右子树中。

时间复杂度 O(n)，空间复杂度 O(n)（递归深度）。

若是二叉搜索树（BST），可利用大小关系优化：

function lowestCommonAncestorBST(root, p, q) {
  while (root) {
    if (p.val < root.val && q.val < root.val) root = root.left;
    else if (p.val > root.val && q.val > root.val) root = root.right;
    else return root;
  }
}

**评分维度**：
- 递归思路（30%）
- 代码正确性（30%）
- 复杂度分析（20%）
- BST 优化（20%）

**常见错误**：
- 递归终止条件写错（遗漏 root === p 或 root === q 的情况）
- 返回值逻辑混乱（不知道 left && right 时返回什么）
- 混淆普通二叉树和 BST 的解法

**扩展追问**：
- 如果树中节点有指向父节点的指针，如何求 LCA？

### 10. 堆（Heap）与 Top K 问题

**问题描述**：给定一个未排序的整数数组，找出其中最大的 K 个数（或最小的 K 个数）。要求分析不同解法的时间复杂度。

**参考答案**：

Top K 问题有几种常见解法：

**解法一：全局排序**
将数组排序后取前 K 个。时间复杂度 O(n log n)，空间复杂度 O(1)。不能处理海量数据。

**解法二：最小堆（最适合海量数据）**
维护一个大小为 K 的最小堆，遍历数组：若堆未满则直接插入，若当前元素大于堆顶则替换堆顶并堆化。遍历结束后堆中即为最大的 K 个数。

class MinHeap {
  constructor() { this.heap = []; }
  size() { return this.heap.length; }
  peek() { return this.heap[0]; }
  push(val) {
    this.heap.push(val);
    this._siftUp(this.heap.length - 1);
  }
  pop() {
    const top = this.heap[0];
    const last = this.heap.pop();
    if (this.heap.length) {
      this.heap[0] = last;
      this._siftDown(0);
    }
    return top;
  }
  _siftUp(i) {
    while (i > 0) {
      const p = (i - 1) >> 1;
      if (this.heap[p] <= this.heap[i]) break;
      [this.heap[p], this.heap[i]] = [this.heap[i], this.heap[p]];
      i = p;
    }
  }
  _siftDown(i) {
    const n = this.heap.length;
    while (true) {
      let smallest = i;
      const l = 2 * i + 1, r = 2 * i + 2;
      if (l < n && this.heap[l] < this.heap[smallest]) smallest = l;
      if (r < n && this.heap[r] < this.heap[smallest]) smallest = r;
      if (smallest === i) break;
      [this.heap[i], this.heap[smallest]] = [this.heap[smallest], this.heap[i]];
      i = smallest;
    }
  }
}

function topK(nums, k) {
  const heap = new MinHeap();
  for (const n of nums) {
    if (heap.size() < k) { heap.push(n); }
    else if (n > heap.peek()) { heap.pop(); heap.push(n); }
  }
  return heap.heap;
}

时间复杂度 O(n log k)，空间复杂度 O(k)。适合处理流式数据和海量数据（无法一次性加载到内存）。

**解法三：快速选择（QuickSelect）**
基于快速排序的 partition 思想，平均时间复杂度 O(n)，最坏 O(n^2)。原地操作，空间 O(1)。

**评分维度**：
- 思路阐述（30%）
- 堆实现正确性（30%）
- 复杂度分析（20%）
- 场景匹配（20%）

**常见错误**：
- 求 Top K 大用最大堆而非最小堆（正确做法是用最小堆保留大值）
- 堆化操作（siftUp/siftDown）实现错误
- 忽略边界情况（k 为 0 或大于数组长度）

### 11. 滑动窗口最大值

**问题描述**：给定一个数组 nums 和滑动窗口大小 k，请找出所有滑动窗口里的最大值。例如输入 [1,3,-1,-3,5,3,6,7] 和 k=3，输出 [3,3,5,5,6,7]。

**参考答案**：

使用双端队列（Deque）维护窗口内可能成为最大值的元素索引。队列保持递减顺序，保证队首始终是当前窗口的最大值。

function maxSlidingWindow(nums, k) {
  const result = [];
  const deque = [];
  for (let i = 0; i < nums.length; i++) {
    // 移除超出窗口范围的元素
    while (deque.length && deque[0] < i - k + 1) deque.shift();
    // 移除队列中所有比当前元素小的元素（它们不可能再成为最大值）
    while (deque.length && nums[deque[deque.length - 1]] < nums[i]) deque.pop();
    deque.push(i);
    // 从第一个完整窗口开始记录结果
    if (i >= k - 1) result.push(nums[deque[0]]);
  }
  return result;
}

时间复杂度 O(n)（每个元素最多入队和出队一次），空间复杂度 O(k)。

- 算法思路（30%）

- 复杂度分析（20%）

### 12. 两数之和及其变体

**问题描述**：给定一个整数数组 nums 和一个目标值 target，请在数组中找出和为目标值的两个数，返回它们的下标。假设每种输入只有一种答案。

**参考答案**：

**经典解法：哈希表**

function twoSum(nums, target) {
  const map = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) {
      return [map.get(complement), i];
    }
    map.set(nums[i], i);
  }
  return [];
}

时间复杂度 O(n)，空间复杂度 O(n)。

**变体一：三数之和（Three Sum）**

找出数组中所有和为 0 的三元组（不重复）。先排序，再用双指针：

function threeSum(nums) {
  nums.sort((a, b) => a - b);
  const result = [];
  for (let i = 0; i < nums.length - 2; i++) {
    if (i > 0 && nums[i] === nums[i - 1]) continue;
    let left = i + 1, right = nums.length - 1;
    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];
      if (sum === 0) {
        result.push([nums[i], nums[left], nums[right]]);
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;
        left++; right--;
      } else if (sum < 0) left++;
      else right--;
    }
  }
  return result;
}

时间复杂度 O(n^2)，空间复杂度 O(log n)（排序空间）或 O(n)。

**变体二：四数之和**：在三数之和的基础上再套一层循环，O(n^3)。

function merge(left, right) {
  const res = [];
  while (left.length && right.length) {
    res.push(left[0] <= right[0] ? left.shift() : right.shift());
  }
  return [...res, ...left, ...right];
}

class VirtualList {
  constructor(container, items, itemHeight) {
    this.container = container;
    this.items = items;
    this.itemHeight = itemHeight;
    this.visibleCount = Math.ceil(container.clientHeight / itemHeight) + 2;
    this.startIndex = 0;
    this.endIndex = this.visibleCount;
    this.totalHeight = items.length * itemHeight;
    this.render();
    container.addEventListener("scroll", () => this.onScroll());
  }
  onScroll() {
    this.startIndex = Math.floor(this.container.scrollTop / this.itemHeight);
    this.endIndex = this.startIndex + this.visibleCount;
    this.render();
  }
  render() {
    const visibleItems = this.items.slice(this.startIndex, this.endIndex);
    this.container.innerHTML = "";

    ul.style.paddingTop = (this.startIndex * this.itemHeight) + "px";

      li.style.height = this.itemHeight + "px";
      ul.appendChild(li);
    }
    this.container.appendChild(ul);
  }

### 15. 如何检测有向图中的环？

**问题描述**：给定一个有向图，检测其中是否存在环。请说明至少两种解法及其应用场景。

**参考答案**：

**解法一：DFS + 三色标记法**

const WHITE = 0, GRAY = 1, BLACK = 2;

function hasCycle(graph) {
  const colors = new Array(graph.length).fill(WHITE);
  function dfs(node) {
    if (colors[node] === GRAY) return true;
    if (colors[node] === BLACK) return false;
    colors[node] = GRAY;
    for (const neighbor of graph[node]) {
      if (dfs(neighbor)) return true;
    }
    colors[node] = BLACK;
    return false;
  }
  for (let i = 0; i < graph.length; i++) {
    if (dfs(i)) return true;
  }
  return false;
}

三色含义：白色为未访问，灰色为正在访问（本路径中），黑色为已访问完成。遇到灰色节点说明存在环。

时间复杂度 O(V + E)，空间复杂度 O(V)。

**解法二：拓扑排序（Kahn 算法）**

function hasCycleTopo(graph) {
  const indegree = new Array(graph.length).fill(0);
  for (const neighbors of graph) {
    for (const v of neighbors) indegree[v]++;
  }
  const queue = [];
  for (let i = 0; i < indegree.length; i++) {
    if (indegree[i] === 0) queue.push(i);
  }
  let count = 0;
  while (queue.length) {
    const u = queue.shift();
    count++;
    for (const v of graph[u]) {
      indegree[v]--;
      if (indegree[v] === 0) queue.push(v);
    }
  }
  return count !== graph.length;
}

应用场景：依赖检测（npm 依赖、模块加载）、死锁检测、课程安排（能否完成所有课程）。

### 16. 基于 Trie 的前端路由匹配

**问题描述**：设计一个路由系统，支持注册路径模式（如 /user/:id、/user/:id/profile）并根据请求路径匹配对应的处理函数。请使用 Trie（前缀树）实现。

**参考答案**：

Trie（前缀树/字典树）是一种多叉树结构，适合字符串前缀匹配。在前端路由中，将路径按段（segment）拆分后构建 Trie，实现高效匹配。

class TrieNode {
  constructor() {
    this.children = new Map();
    this.handler = null;
    this.paramName = null;
  }
}

class Router {
  constructor() {
    this.root = new TrieNode();
  }
  addRoute(path, handler) {
    const segments = path.split("/").filter(Boolean);
    let node = this.root;
    for (const seg of segments) {
      if (seg.startsWith(":")) {
        if (!node.children.has(":")) node.children.set(":", new TrieNode());
        node = node.children.get(":");
        node.paramName = seg.slice(1);
      } else {
        if (!node.children.has(seg)) node.children.set(seg, new TrieNode());
        node = node.children.get(seg);
      }
    }
    node.handler = handler;
  }
  match(path) {
    const segments = path.split("/").filter(Boolean);
    const params = {};
    let node = this.root;
    for (const seg of segments) {
      if (node.children.has(seg)) {
        node = node.children.get(seg);
      } else if (node.children.has(":")) {
        node = node.children.get(":");
        params[node.paramName] = seg;
      } else {
        return null;
      }
    }
    return node.handler ? { handler: node.handler, params } : null;
  }
}

**进阶版：不限交易次数**

function maxProfitII(prices) {

    if (prices[i] > prices[i - 1]) {
      profit += prices[i] - prices[i - 1];
    }
  }
  return profit;

**高阶版：最多两次交易（DP）**

function maxProfitIII(prices) {
  let buy1 = -prices[0], sell1 = 0;
  let buy2 = -prices[0], sell2 = 0;
  for (let i = 1; i < prices.length; i++) {

### 18. 岛屿数量（DFS / BFS）

**参考答案**：

遍历整个网格，每遇到一个 "1" 就将其作为起点进行 DFS/BFS 搜索，将所有相连的 "1" 标记为已访问（置为 "0"），岛屿数量加一。

**DFS 解法**：

function numIslands(grid) {
  if (!grid || !grid.length) return 0;
  const rows = grid.length, cols = grid[0].length;
  let count = 0;
  function dfs(r, c) {
    if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] === "0") return;
    grid[r][c] = "0";
    dfs(r + 1, c);
    dfs(r - 1, c);
    dfs(r, c + 1);
    dfs(r, c - 1);
  }
  for (let r = 0; r < rows; r++) {
    for (let c = 0; c < cols; c++) {
      if (grid[r][c] === "1") {
        count++;
        dfs(r, c);
      }
    }
  }
  return count;
}

时间复杂度 O(m * n)，空间复杂度 O(m * n)（递归调用栈深度）。

**BFS 解法**：使用队列代替递归，避免栈溢出风险。同样置 "1" 为 "0" 后入队，扩散搜索相邻节点。

### 20. 合并 K 个有序链表

**问题描述**：给定 K 个有序链表的头节点数组，将所有链表合并为一个有序链表并返回。请分析不同解法的时间复杂度。

**参考答案**：

**解法一：顺序合并**
每次合并两个链表，重复 K-1 次。若第 i 次合并时总长度为 i*n，总时间复杂度 O(K^2 * n)。

**解法二：分治合并（类似归并排序）**

function mergeKLists(lists) {
  if (!lists || !lists.length) return null;
  function mergeTwo(l1, l2) {
    const dummy = new ListNode(0);
    let curr = dummy;
    while (l1 && l2) {
      if (l1.val < l2.val) { curr.next = l1; l1 = l1.next; }
      else { curr.next = l2; l2 = l2.next; }
      curr = curr.next;
    }
    curr.next = l1 || l2;
    return dummy.next;
  }
  function merge(l, r) {
    if (l === r) return lists[l];
    if (l > r) return null;
    const mid = (l + r) >> 1;
    return mergeTwo(merge(l, mid), merge(mid + 1, r));
  }
  return merge(0, lists.length - 1);
}

时间复杂度 O(Kn * log K)，空间复杂度 O(log K)（递归栈）。

**解法三：最小堆（优先队列）**

function mergeKLists(lists) {
  const dummy = new ListNode(0);
  let curr = dummy;
  // 使用最小堆，按节点值排序
  const heap = new MinHeap((a, b) => a.val - b.val);
  for (const head of lists) {
    if (head) heap.push(head);
  }
  while (heap.size()) {
    const node = heap.pop();
    curr.next = node;
    curr = curr.next;
    if (node.next) heap.push(node.next);
  }
  return dummy.next;
}

时间复杂度 O(Kn * log K)，空间复杂度 O(K)。适合流式处理场景。

**评分维度**：
- 多种解法对比（30%）
- 分治实现（25%）
- 堆实现（25%）
- 复杂度分析（20%）

**常见错误**：
- 未处理链表数组为空或包含空链表的情况
- 堆的比较器写反（用了最大堆）
- 分治中递归合并的两个链表顺序写错

**扩展追问**：
- 如果链表数据量极大无法全部放入内存，如何处理？（外部归并）
- 如何合并 K 个有序数组？
- 最小堆的堆化过程如何实现？

## 四、面试技巧与建议

### 1. 解题四步法

在面试中遇到算法题，推荐按以下流程组织回答：

1. **确认题意**（2-3 分钟）
   - 重复问题确保理解正确
   - 确认输入输出的类型、范围、边界条件
   - 举例验证（小规模正常情况 + 边界情况）

2. **思路阐述**（3-5 分钟）
   - 先给出暴力解，指出其复杂度
   - 分析如何优化，陈述最优解的核心思路
   - 与面试官讨论确认后再开始编码

3. **编码实现**（10-15 分钟）
   - 边写边注释关键逻辑
   - 保持代码风格清晰，变量命名有意义
   - 注意边界处理（空输入、单元素、大数值）

4. **验证复盘**（3-5 分钟）
   - 用测试用例走查代码（正常 + 边界）
   - 分析时间和空间复杂度
   - 讨论可能的优化方向和扩展问题

### 2. 常见误区

- **急于编码**：思路未清晰就开始写代码，容易写出漏洞百出的实现
- **沉默编程**：面试官希望看到你的思考过程，不要默默写代码
- **忽视复杂度**：只写对了但分析不出复杂度是常见扣分点
- **死记硬背**：背题不如理解思路，面试考的是思维过程而非标准答案
- **忽略前端场景**：前端面试中要主动将算法与前端工程实践结合

### 3. 前端算法面试的独特之处

- **关注交互场景**：虚拟列表、DOM Diff、路由匹配等前端特有的算法问题要重点准备
- **事件循环与异步**：宏任务/微任务、事件循环机制是高频考点
- **性能敏感**：页面渲染性能、大量数据处理是前端面试中的加分方向
- **浏览器 API**：可利用 Map、Set、WeakMap 等原生数据结构简化实现

### 4. 推荐练习计划

- **基础阶段**（2 周）：掌握数组、链表、栈、队列、哈希表的操作和复杂度
- **进阶阶段**（3 周）：二叉树遍历、堆、排序、双指针、滑动窗口、BFS/DFS
- **高级阶段**（3 周）：动态规划、图论、Trie、复杂场景设计
- **冲刺阶段**（1 周）：限时模拟面试，锻炼在规定时间内解题的能力
