# 前端架构师知识图谱与进阶规划

> 本文档用于指导前端工程师从"能写页面"逐步成长为"前端架构师"，覆盖知识体系、能力模型与阶段性进阶路线。
> 本规划追求尽善尽美：即使是初级学员，也能按阶段循序渐进，最终具备架构师所需的技术深度、系统广度和战略思维。

## 三、进阶路线规划

> 以下阶段以**能力里程碑**为核心，而非严格年限。能力强的人可以加速跨越，基础薄弱的人需要扎实沉淀。

### 📍 阶段一：高级前端工程师

**目标**：从"能写页面"到"能写工程"。

- [ ] 深入掌握 JavaScript / TypeScript，能编写健壮、可维护的业务代码
- [ ] 熟悉至少一个主流框架的源码级原理
- [ ] 能独立完成组件库、工具库的封装
- [ ] 掌握前端工程化基本配置（构建、lint、测试、CI）
- [ ] 具备基本的性能优化意识和实践能力
- [ ] 能阅读和理解简单的后端接口与 BFF 设计

**通关标准**：独立搭建一个组件库 + 文档站点，并完成 CI/CD 流水线配置。

### 📍 阶段二：前端专家

**目标**：解决复杂系统问题，带动团队。

- [ ] 主导中大型项目的架构设计与技术选型
- [ ] 建立性能优化的体系化方法论
- [ ] 熟悉微前端、跨端、SSR、AI 工程化等高级技术方案
- [ ] 制定团队开发规范、主导 Code Review、推动代码质量
- [ ] 能设计合理的数据流与状态管理方案
- [ ] 能建立基础的可观测性体系（监控、埋点、告警）
- [ ] 定期进行技术分享与培训，提升团队整体水平

**通关标准**：主导一个中型项目的完整架构设计，输出 ADR，并推动落地。

### 📍 阶段三：前端架构师

**目标**：从"技术实现"到"技术战略"。

- [ ] 站在业务视角进行技术选型与演进规划
- [ ] 设计可扩展、可维护的前端技术体系
- [ ] 推动工程效率、代码质量、系统性能的全面提升
- [ ] 具备团队管理能力与跨职能影响力
- [ ] 关注行业趋势，持续引入新技术并评估落地价值
- [ ] 能制定 1-3 年技术演进路线图，并获得业务方认可

**通关标准**：制定并推动团队技术演进路线图，至少完成一个重大架构升级项目。

## 五、关键行动清单

1. **每周**阅读一篇源码或技术深度文章。
2. **每月**完成一个小型实践项目或技术分享。
3. **每季度**复盘一次个人技术成长，调整学习重点。
4. **每年**参与或主导一个中大型项目，积累架构经验。
5. **持续**跟踪技术趋势，更新个人技术雷达。

## 六、总结

> 前端架构师的核心竞争力不是"会多少框架"，而是**在业务、技术、团队之间找到最优解的能力**。
