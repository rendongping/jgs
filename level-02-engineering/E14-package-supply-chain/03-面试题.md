# 前端包管理与供应链安全面试题

> 模拟真实面试场景，训练对包管理和供应链安全的思考与表达。

---

## 一、基础题

### 1. npm 的语义化版本规范是什么？

**参考答案**：

版本号格式 MAJOR.MINOR.PATCH：
- MAJOR：不兼容的 API 修改。
- MINOR：向下兼容的功能新增。
- PATCH：向下兼容的问题修复。

**评分维度**：
- 格式解释（50%）
- 三个字段含义（30%）
- 前缀 ^/~ 区别（20%）

---

### 2. 什么是幽灵依赖？如何避免？

**参考答案**：

幽灵依赖是指项目直接使用了未在 package.json 中声明的间接依赖。npm/yarn 的扁平化结构容易导致这个问题。pnpm 的严格 node_modules 结构可以避免。

**评分维度**：
- 定义准确（40%）
- 产生原因（30%）
- 解决方案（30%）

---

### 3. lockfile 是否应该提交到版本控制？为什么？

**参考答案**：

应该提交。lockfile 锁定确切版本，保证团队成员和 CI/CD 环境安装的依赖一致，避免"我本地可以跑"的问题。

**评分维度**：
- 结论（30%）
- 可复现性（40%）
- 团队协作（30%）

---

## 二、进阶题

### 4. 如何评估是否引入一个新的 npm 包？

**参考答案**：

- 功能是否必要，是否有更轻量替代。
- 维护活跃度、issue 响应、发布频率。
- 包体积和依赖数量。
- 许可证是否合规。
- 安全记录和审计结果。

**评分维度**：
- 功能评估（25%）
- 维护活跃度（25%）
- 体积与依赖（20%）
- 许可与安全（20%）
- 团队影响（10%）

---

### 5. 供应链攻击有哪些常见形式？

**参考答案**：

- 恶意包：包中植入后门或窃取代码。
- 依赖混淆：上传与内部包同名的公共包。
- 版本劫持：篡改已发布版本或 typosquatting。
- 安装脚本攻击：利用 postinstall 执行恶意命令。

**评分维度**：
- 攻击类型覆盖（50%）
- 原理说明（30%）
- 防护措施（20%）

---

## 三、高级题

### 6. 设计一个企业级依赖安全治理体系。

**参考答案要点**：

1. 引入审批：白名单 + 自动化扫描。
2. 私有 registry：统一来源，禁止直接从公共源安装。
3. 持续审计：CI 中集成 npm audit/Snyk/SBOM 检查。
4. 漏洞响应：分级处理、SLA、紧急修复流程。
5. 定期清理：移除无用依赖，降低攻击面。

**评分维度**：
- 引入管控（25%）
- 来源控制（20%）
- 持续审计（20%）
- 响应机制（20%）
- 可持续性（15%）

---

### 7. Monorepo 中如何做好依赖版本管理？

**参考答案要点**：

1. 公共依赖提升到 root，减少重复。
2. 内部包使用 workspace 协议。
3. 统一版本策略，避免多版本冲突。
4. 使用 Changesets 管理版本和 changelog。
5. 构建依赖图，检测循环依赖和无用依赖。

**评分维度**：
- 公共依赖管理（25%）
- 内部包协议（20%）
- 版本策略（20%）
- 工具选择（15%）
- 风险控制（20%）
