# E14 前端包管理与供应链安全

> 目标：掌握前端包管理原理、依赖治理和供应链安全防护，降低第三方依赖带来的风险。

---

## 核心要点（TL;DR）

- 现代前端项目依赖成百上千个 npm 包，包管理是工程化的基础能力。
- npm/yarn/pnpm 的依赖解析、安装、锁定机制各有差异。
- 供应链攻击（如恶意包、依赖混淆）已成为前端重大安全威胁。
- 依赖治理需要制度化：审批、审计、更新、替换、SBOM。
- lockfile 是构建可复现性的关键，必须纳入版本控制。

---

## 1. 包管理器原理

### 1.1 npm/yarn/pnpm 对比

| 特性 | npm | yarn | pnpm |
|------|-----|------|------|
| 安装速度 | 中等 | 快 | 最快 |
| 磁盘占用 | 大 | 大 | 小（硬链接） |
| node_modules 结构 | 扁平 | 扁平 | 严格 |
| workspace 支持 | 有 | 有 | 优秀 |

### 1.2 依赖解析

- 语义化版本：^、~、>=、< 等。
- 依赖树：可能产生版本冲突和幽灵依赖。
- pnpm 的严格结构避免了幽灵依赖。

### 1.3 lockfile

- package-lock.json、yarn.lock、pnpm-lock.yaml。
- 锁定确切版本，保证构建可复现。
- 应提交到版本控制。

## 2. 依赖治理

### 2.1 依赖评估

-  stars / 维护活跃度 / issue 响应速度。
-  包体积 / 依赖数量 / 许可协议。
-  是否有替代方案。

### 2.2 依赖审计

- `npm audit`：扫描已知漏洞。
- Snyk、Dependabot、OSV。
- 建立漏洞修复 SLA。

### 2.3 依赖升级

- 小版本自动升级，大版本人工评估。
- 使用 Renovate/Dependabot 自动提 PR。
- 升级前跑回归测试。

### 2.4 私有仓库

- Verdaccio、Nexus、Artifactory。
- 统一内部包发布和权限管理。

---

## 3. 供应链安全

### 3.1 常见攻击

| 攻击类型 | 说明 | 防护 |
|----------|------|------|
| 恶意包 | 包中植入后门 | 审计、沙箱、来源审查 |
| 依赖混淆 | 上传与内部包同名的公共包 | 命名空间、私有 registry |
| 版本劫持 | 篡改已发布版本 | lockfile、签名验证 |
| typosquatting | 仿冒知名包名 | 安装前核对 |

### 3.2 安全实践

- 固定版本 + lockfile。
- 使用私有 registry 和 scoped packages。
- 运行前静态扫描 + 动态沙箱。
- 生成 SBOM（软件物料清单）。
- 最小权限原则：限制安装脚本执行。

### 3.3 SBOM

- 记录项目所有依赖及其来源。
- 格式：SPDX、CycloneDX。
- 用途：合规、审计、漏洞响应。

---

## 4. Monorepo 中的依赖管理

- workspace 协议。
- 统一版本策略。
- Changesets 管理版本和 changelog。
- 避免循环依赖。

---

## 常见误区

| 误区 | 正确理解 |
|------|---------|
| 直接 copy 网上的安装命令 | 应核对包名、版本和来源 |
| lockfile 不需要提交 | lockfile 是构建可复现的关键 |
| 只要功能正常就不用升级 | 旧依赖可能存在安全漏洞 |
| 供应链安全只靠安全团队 | 每个开发者都应有安全意识 |
