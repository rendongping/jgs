# 实时与协同架构面试题

> 模拟真实面试场景，训练对实时通信、长连接治理、协同编辑的理解与表达。

---

## 一、基础题

### 1. WebSocket 和 SSE 有什么区别？分别适用什么场景？

**参考答案**：

| 特性 | WebSocket | SSE |
|------|-----------|-----|
| 方向 | 双向 | 服务端 → 客户端 |
| 协议 | ws/wss | HTTP |
| 自动重连 | 需手动实现 | 浏览器原生支持 |
| 适用 | 聊天、游戏、协同编辑 | 推送、行情、通知 |

**评分维度**：
- 方向区别（30%）
- 协议区别（20%）
- 场景举例（40%）
- 重连机制（10%）

---

### 2. 如何设计 WebSocket 重连机制？

**参考答案**：

- 监听 onclose/onerror 触发重连。
- 使用指数退避控制重连频率。
- 设置最大重试次数或无限重试。
- 断线期间缓存待发送消息。
- 重连后恢复房间状态、补发未确认消息。

**评分维度**：
- 触发条件（20%）
- 退避策略（30%）
- 消息缓存（25%）
- 状态恢复（25%）

---

### 3. 什么是 CRDT？与 OT 有什么区别？

**参考答案**：

CRDT 是具有数学特性的数据结构，允许多个副本独立修改并最终一致。OT 需要中央服务器对操作进行变换。

CRDT 更适合离线优先、去中心化场景；OT 适合强一致、中心化场景。

**评分维度**：
- 解释 CRDT（40%）
- 解释 OT（30%）
- 选型对比（30%）

---

## 二、进阶题

### 4. WebSocket 集群如何实现跨节点广播？

**参考答案**：

- 使用 Redis Pub/Sub、RabbitMQ 等消息中间件。
- 每个 WebSocket 节点订阅房间频道。
- 消息发送到中间件，所有订阅节点转发给本地连接。
- 或使用 sticky session 将同一房间用户路由到同一节点。

**评分维度**：
- 中间件方案（40%）
- 订阅机制（30%）
- 扩展性考虑（30%）

---

### 5. 实时系统的可观测性应关注哪些指标？

**参考答案**：

- 连接数、在线用户数。
- 消息延迟（端到端）。
- 断线率、重连成功率。
- 消息丢失率/重复率。
- 服务端资源使用（CPU、内存、带宽）。

**评分维度**：
- 连接相关指标（30%）
- 消息质量指标（40%）
- 资源指标（20%）
- 告警机制（10%）

---

## 三、高级题

### 6. 设计一个支持离线编辑的协同文档系统。

**参考答案要点**：

1. 客户端使用 CRDT（如 Yjs）管理文档。
2. 本地持久化操作日志。
3. 联网后同步操作，CRDT 自动解决冲突。
4. 服务端持久化操作历史，支持版本回溯。
5. 设计心跳和同步确认机制。

**评分维度**：
- CRDT 选择（25%）
- 离线存储（25%）
- 同步机制（25%）
- 版本与可观测性（25%）

---

### 7. 设计一个百万级在线直播弹幕系统。

**参考答案要点**：

1. WebSocket 网关集群 + 负载均衡。
2. 消息队列削峰（Kafka/Pulsar）。
3. 按房间分片，减少广播范围。
4. 弹幕过滤、频率限制、合并相似内容。
5. 多级缓存和降级策略。
6. 实时监控连接数、消息延迟、丢包率。

**评分维度**：
- 接入层设计（25%）
- 消息流转（25%）
- 限流与降级（25%）
- 可观测性（25%）
