## 参考答案

### 第 1 题

::: details 查看答案与解析
**答案：C**

WebGL 利用 GPU 并行渲染，适合大规模 3D 场景。
:::

### 第 2 题

::: details 查看答案与解析
**答案：B**

SVG 是矢量图形，缩放不失真，每个图形都是 DOM 节点，易于交互。
:::

### 第 3 题

::: details 查看答案与解析
**答案：B**

Canvas 性能瓶颈通常是绘制调用过多，应批量绘制。
:::

### 第 4 题

::: details 查看答案与解析
**答案：B**

10 万点散点图应使用 Canvas 批量绘制或 WebGL。
:::

### 第 5 题

::: details 查看答案与解析
**答案：B**

虚拟化只渲染可视区域，减少绘制量。
:::

### 第 6 题

::: details 查看答案与解析
**参考思路**：
- 5000 个可点击节点，SVG 仍可行，但需分层或虚拟化。
- 若交互复杂，可用 Canvas + 自定义命中检测。
- 若节点数量继续增长，考虑 WebGL。
:::

### 第 7 题

::: details 查看答案与解析
**参考思路**：
- 数据降采样（LTTB）。
- 使用 Canvas 或 WebGL。
- 分层渲染：坐标轴和数据层分离。
- 交互时显示原始数据局部细节。
:::

### 第 8 题

::: details 查看答案与解析
**参考思路**：
- 抽象渲染接口：Renderer。
- 实现 SVGRenderer、CanvasRenderer、WebGLRenderer。
- 上层图表组件依赖 Renderer 接口。
- 根据数据量和配置自动选择或手动指定后端。
:::

### 第 9 题

**参考选型**：
- 折线图/饼图：ECharts 或 AntV G2。
- 地图：结合地图服务 + Canvas/WebGL 渲染 POI。
- 理由：开发效率高、文档完善、社区活跃。

### 第 10 题

::: details 查看答案与解析
**参考思路**：
- 地图按缩放级别切分为瓦片。
- 只加载可视区域瓦片。
- POI 按四叉树或网格索引组织。
- 视口外 POI 不渲染，聚合相近 POI。
:::

### 第 11 题

::: details 查看答案与解析
**参考思路**：
- 维护图形对象列表，记录位置、形状、边界。
- 监听鼠标事件，将坐标转换为 Canvas 坐标。
- 使用几何检测（点是否在矩形/圆/多边形内）。
- 优化：空间索引（R-tree、网格）加速命中检测。
:::
