# A10 前端可视化与图形架构

> 目标：掌握前端可视化技术选型、渲染原理和图形架构设计，能够构建高性能数据可视化系统。

---

## 核心要点（TL;DR）

- 可视化选型取决于数据规模、交互复杂度、渲染性能和团队能力。
- SVG 适合静态、可交互、DOM 操作友好的场景；Canvas 适合像素级控制和大量图形；WebGL 适合大规模 3D/2D 渲染。
- 大数据量渲染的核心是减少绘制调用和 DOM 节点，常用手段包括分层、瓦片、LOD、虚拟化。
- 图形架构需要抽象渲染层、数据层、交互层，降低与具体渲染技术的耦合。

---

## 1. 可视化技术选型

### 1.1 SVG

- 矢量图形，缩放不失真。
- 每个图形都是 DOM 节点，便于事件绑定和 CSS 样式。
- 适合：折线图、柱状图、饼图、流程图、地图。
- 局限：节点过多时性能下降。

### 1.2 Canvas 2D

- 位图渲染，通过 JavaScript 直接绘制像素。
- 适合：大量数据点、复杂动画、游戏、图表库底层。
- 局限：事件处理需自己实现命中检测。

### 1.3 WebGL / WebGPU

- 利用 GPU 并行渲染。
- 适合：大规模散点图、3D 场景、实时渲染、粒子效果。
- 学习曲线陡峭。

### 1.4 选型对比

| 技术 | 数据量 | 交互 | 性能 | 复杂度 |
|------|--------|------|------|--------|
| SVG | 中小 | 强 | 中 | 低 |
| Canvas | 大 | 中 | 高 | 中 |
| WebGL | 超大 | 中 | 极高 | 高 |

## 2. 常见可视化库

- **ECharts**：功能丰富，适合企业级图表。
- **D3.js**：灵活性高，适合自定义可视化。
- **AntV**：蚂蚁生态，G2/G6/L7 覆盖图表、图、地图。
- **Three.js**：3D 渲染封装。
- **Deck.gl**：大规模地理数据可视化。
- **PixiJS**：2D 游戏和动画引擎。

---

## 3. 高性能渲染策略

### 3.1 分层渲染

- 将静态背景、动态数据、交互元素分在不同层。
- 只有变化层需要重绘。

### 3.2 数据降采样

- 大数据量时抽取关键数据点。
- 常用算法：LTTB（Largest Triangle Three Buckets）。

### 3.3 虚拟化

- 只渲染可视区域数据。
- 应用于地图瓦片、长列表、时间轴。

### 3.4 离屏渲染

- 在 Worker 或离屏 Canvas 中预计算复杂图形。

### 3.5 GPU 加速

- 使用 transform、will-change、WebGL 实例化渲染。

---

## 4. 图形架构设计

### 4.1 分层架构

```
应用层
├── 图表组件层
├── 渲染引擎层（SVG/Canvas/WebGL 抽象）
├── 数据处理层
└── 交互事件层
```

### 4.2 数据到图形的映射

- 数据 → 比例尺（Scale）→ 图形属性（位置、颜色、大小）→ 渲染。

### 4.3 交互设计

- 缩放、平移、刷选、提示框、图例、下钻。
- 事件系统需要支持命中检测和坐标转换。

---

## 5. 前端可视化最佳实践

- 避免在 SVG 中渲染超过 1 万个节点。
- Canvas 中批量绘制，减少状态切换。
- 动画使用 requestAnimationFrame。
- 响应式布局监听容器尺寸变化。
- 可访问性：提供文字替代和键盘操作。

---

## 常见误区

| 误区 | 正确理解 |
|------|---------|
| 所有图表都用 ECharts | 根据场景选型，复杂自定义需 D3/WebGL |
| 可视化只关心美观 | 数据准确性、性能、可访问性同样重要 |
| 一次性渲染所有数据 | 大数据量需要降采样和虚拟化 |
| 忽略移动端性能 | 移动端 GPU 和内存有限 |
