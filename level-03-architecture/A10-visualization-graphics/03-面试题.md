## 一、基础题

### 1. SVG 和 Canvas 的主要区别是什么？

**参考答案**：

SVG 是矢量图形，基于 DOM，每个图形都是 DOM 节点，适合交互和样式控制；Canvas 是位图渲染，通过脚本绘制像素，适合大量图形和高频动画。

核心区别对比：
- **渲染方式**：SVG 保留图形对象（保留模式），Canvas 立即绘制像素（立即模式）。
- **性能**：SVG 在节点数超过数千时性能急剧下降；Canvas 仅与像素数相关，适合数万以上的图形。
- **交互**：SVG 原生支持 DOM 事件；Canvas 需要手动实现命中检测。
- **缩放**：SVG 矢量缩放不失真；Canvas 位图缩放会模糊。
- **动画**：SVG 支持 CSS 动画；Canvas 通过 requestAnimationFrame 驱动。

**评分维度**：
- 渲染方式（40%）
- 性能差异（30%）
- 交互差异（20%）
- 适用场景（10%）

### 4. Canvas 2D API 基础：绘制原理与状态管理

**参考答案**：

Canvas 2D 通过 CanvasRenderingContext2D 提供基于路径的绘制模型。

核心概念：
- **绘制状态栈**：ctx.save() 压入当前状态，ctx.restore() 恢复上一级状态。
- **路径系统**：beginPath() -> moveTo/lineTo/arc/bezierCurveTo -> closePath() -> fill/stroke。
- **变换矩阵**：translate/rotate/scale/setTransform 影响所有后续绘制。
- **像素操作**：getImageData/putImageData 提供逐像素控制。
- **离屏渲染**：OffscreenCanvas 或辅助 canvas 预渲染。

**评分维度**：
- 状态管理理解（30%）
- API 熟练度（25%）
- 性能优化意识（25%）
- 离屏渲染应用（20%）

**常见错误**：
1. 忘记调用 beginPath() 导致路径堆积。
2. save/restore 不配对，状态泄漏。
3. 动画循环中频繁创建 Canvas 对象。
4. 混淆 canvas.width/height 与 CSS 尺寸。

### 6. D3.js 数据绑定与 Data Join 机制

**参考答案**：

D3 的 Data Join 是将数据与 DOM 元素绑定的模式，核心是 enter-update-exit 三段式。

- **Data Join**：selection.data(data, keyFunction) 绑定数据。
- **Enter**：数据 > DOM 时，enter() 返回占位符，通过 append 创建新元素。
- **Update**：数据 === DOM 时，直接更新已有元素。
- **Exit**：数据 < DOM 时，exit() 返回多余元素，remove 删除。
- **Merge**：merge() 合并 enter 和 update 统一设置属性。

**评分维度**：
- 数据绑定原理（30%）
- Enter-Update-Exit 流程（30%）
- Key 函数设计（20%）
- 过渡动画（20%）

**常见错误**：
1. 未提供 key 函数，依赖索引导致增删错乱。
2. 每次更新重新 selectAll 而非复用选择集。
3. 混淆 datum() 和 data()。
4. Enter 和 Update 未分别处理初始属性和过渡属性。

**扩展追问**：

### 7. requestAnimationFrame 与 setInterval 在动画中的选择

**参考答案**：

requestAnimationFrame (rAF) 由浏览器在重绘前调用，与刷新率同步。setInterval 固定间隔执行。

对比：
- **执行时机**：rAF 在重绘前统一执行；setInterval 定时器到期立即执行。
- **帧率同步**：rAF 自动同步显示器；setInterval 可能丢帧或过度绘制。
- **后台行为**：rAF 页面不可见自动暂停；setInterval 持续执行。
- **精度**：rAF 高 vsync 对齐；setInterval 低，受事件队列阻塞。

**评分维度**：
- 机制理解（30%）
- 性能对比（30%）
- 实际应用（20%）
- 边界情况处理（20%）

**常见错误**：
1. setInterval(fn, 1000/60) 无法保证 60fps 精度。
2. rAF 中嵌套 setTimeout 破坏帧同步。
3. 忘记 cancelAnimationFrame 导致内存泄漏。
4. rAF 回调中执行耗时操作导致帧率抖动。

**扩展追问**：
1. rAF ID 与 setTimeout ID 是否共用池？
2. 低端设备上自适应帧率降级策略？

### 11. 实现一个力导向图可视化

**参考答案**：

力导向图通过物理力学系统布局网络节点。

核心算法：
- **斥力**：库仑定律 F = k_r / d^2。
- **引力**：胡克定律 F = k_s * (d - restLength)。
- **阻尼**：避免震荡，通常 0.6~0.9。
- **冷却**：逐步减小力的大小。
- **Barnes-Hut 优化**：四叉树空间划分，O(n^2) 降为 O(n log n)。

**评分维度**：
- 力学模型理解（25%）
- 算法优化（25%）
- 渲染选型（20%）
- 交互实现（15%）
- 性能考量（15%）

**常见错误**：
1. 未实现冷却机制导致持续震荡。
2. 斥力引力参数失衡。
3. 大规模图全量 O(n^2) 计算。
4. 拖拽时未固定节点位置。

**扩展追问**：
1. 如何实现分层布局？
2. Web Worker 中力模拟如何同步渲染线程？

### 12. Canvas 命中检测策略

策略对比：
- **数学计算**：高精度快，适合简单几何。

- **空间分割**：中精度极快，适合海量散点。

颜色索引法：离屏 Canvas 用颜色 ID 绘制图形，鼠标事件读取像素颜色映射回图形。

### 23. VR/AR 数据可视化 (WebXR)

**参考答案**：

WebXR API：requestSession 获取会话，XRReferenceSpace 定义坐标，Three.js 集成渲染。

交互：手柄射线拾取、手势识别、注视交互。

设计原则：视野 15 度内，深度 0.5~5 米，避免用户疲劳。

**评分维度**：
- WebXR API 掌握（25%）

- 交互设计（20%）

1. 未处理晕动症。

4. 对象过多未做遮挡处理。
